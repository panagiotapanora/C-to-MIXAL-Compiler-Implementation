Υλοποίηση εργασίας
Ο μεταγλωττιστής μπορεί να υλοποιηθεί στη γλώσσα προγραμματισμού της προτίμησής σας αρκεί να
έχετε στη διάθεσή σας τα απαραίτητα εργαλεία (π.χ. flex και yacc ή κλώνους για γλώσσες
προγραμματισμού εκτός της C). Τα περισσότερα εργαλεία αυτού του τύπου λειτουργούνε όπως
περιγράφεται στις σημειώσεις/διαφάνειες/σύγγραμμα μαθήματος και όπως φαίνεται από την
υλοποίηση του πρότυπου εκπαιδευτικού μεταγλωττιστή, που ο κώδικάς του (σε γλώσσα C) διατίθεται
στα εργαστηριακά που έχουμε κάνει και έχουν αναρτηθεί στην ιστοσελίδα του μαθήματος στο elearning.

Τις τελευταίες εκδόσεις των εργαλείων flex και yacc σε εκτελέσιμη μορφή για windows μπορείτε να
τις βρείτε στη διεύθυνση http://gnuwin32.sourceforge.net/packages.html. Tutorial για τη χρήση τους
μπορείτε να βρείτε στη διεύθυνση https://developer.ibm.com/tutorials/au-lexyacc/ και στη
διεύθυνση https://arcb.csc.ncsu.edu/~mueller/codeopt/codeopt00/y_man.pdf καθώς και στο
σύγγραμμα του μαθήματος. Ο μεταγλωττιστής που θα κατασκευάσετε θα κάνει μετάφραση στη
γλώσσα της μηχανής MIX, που περιγράφεται παρακάτω.
Λεξική Δομή Γλώσσας Προγραμματισμού
1. Όλα τα σύμβολα της γραμματικής που ακολουθεί με πεζούς χαρακτήρας αναπαριστούν
κυριολεκτικές τιμές (literals), που είναι δεσμευμένες λέξεις και δεν μπορούν να
χρησιμοποιηθούν ως ονόματα σε μεταβλητές ή μεθόδους.
2. Οι μη αλφαριθμητικοί χαρακτήρες που περικλείονται σε ‘ και ’ αναπαριστούν την
κυριολεκτική τιμή, που αποτελείται μόνο από τους μη αλφαριθμητικούς χαρακτήρες.
3. Η κυριολεκτική τιμή ‘true’ αντιστοιχεί στην τιμή 1, ενώ η ‘false’ αντιστοιχεί στην τιμή 0.
4. Οι υπόλοιπες κυριολεκτικές τιμές είναι οι id και num, που ορίζονται από τις παρακάτω
κανονικές εκφράσεις (γίνεται διάκριση μεταξύ πεζών και κεφαλαίων – το ‘*’ εκφράζει 0 ή
περισσότερες εμφανίσεις, το ‘?’ εκφράζει 0 ή 1 εμφάνιση, το ‘[...]’ εκφράζει διάστημα
χαρακτήρων και το ‘|’ εκφράζει εναλλακτικές):
id = letter (letter | digit | ‘_’)*
num = ‘-’? [1-9] digit*
letter = a | . . . | z | A | . . . | Z
digit = 0 | . . . | 9

5. Τα κενά (διάστημα, αλλαγές γραμμής, tabs) αγνοούνται, αλλά λαμβάνονται υπόψη ως
διαχωριστές μεταξύ των κυριολεκτικών τιμών.
6. Οι χαρακτήρες που συμπεριλαμβάνονται ανάμεσα ‘//’ και την αλλαγή γραμμής
ερμηνεύονται ως σχόλια και γι’ αυτό πρέπει να αγνοούνται.

Κάθε πρόγραμμα της γλώσσας είναι μία σειρά από μηδέν ή περισσότερες δηλώσεις μεθόδων
(ΜΕΤΗ). Ο μοναδικός τύπος που υποστηρίζεται από τη γλώσσα είναι ο int. Κάθε δήλωση μεθόδου
έχει έναν επιστρεφόμενο τύπο (TYPE), μηδέν ή περισσότερες τυπικές παραμέτρους (PARAMS) και
ένα σώμα (BODY). Το σώμα αποτελείται από μηδέν ή περισσότερες δηλώσεις μεταβλητών (DECLS)
και μία ακολουθία εντολών (STMTS). Οι μεταβλητές κατά τη δήλωσή τους μπορεί να

αρχικοποιούνται. Στην ακολουθία εντολών, κάθε εντολή μπορεί να είναι είτε μία εκχώρηση τιμής
(ASSIGN), είτε μία υποθετική εντολή, είτε ένας βρόγχος while, είτε μία return ή break ή ένα block ή
ακόμη και μία κενή εντολή.
Σημασία Γλώσσας Προγραμματισμού
Οι εντολές της γλώσσας έχουν τη συνήθη σημασία. Όμως μία εντολή break θα πρέπει να βρίσκεται
εμφωλευμένη σε έναν ή περισσότερους ένθετους βρόγχους και όταν εκτελείται θα τερματίζει την
εκτέλεση του πλησιέστερου εσωτερικού βρόγχου, στον οποίο βρίσκεται εμφωλευμένη.
Σε ότι αφορά τις εκφράσεις, που βρίσκονται σε συνθήκες, αν αυτές αποτιμώνται σε οποιαδήποτε μη
μηδενική τιμή, τότε θεωρείται ότι είναι true, ενώ αν η τιμή είναι μηδέν, τότε είναι false.
Σχετικά με τη σημασία των προγραμμάτων, ισχύουν επίσης τα παρακάτω:
1. Αφού όλες οι τοπικές μεταβλητές ορίζονται στην αρχή της κάθε μεθόδου, δε χρειάζεται
να απασχοληθείτε με κανόνες εμβέλειας για μεταβλητές.
2. Θα πρέπει να υπάρχει πάντα μία μέθοδος main στο πηγαίο πρόγραμμα.
3. Δεν επιτρέπεται η υπερφόρτωση των μεθόδων.
4. Είναι πιθανό να χρειαστείτε ξεχωριστό πίνακα συμβόλων για κάθε μέθοδο.
Αν ένα πρόγραμμα δεν ικανοποιεί την παραπάνω γραμματική ή δεν ικανοποιεί την περιγραφή της
σημασίας της γλώσσας, τότε ο μεταγλωττιστής σας θα πρέπει να τυπώνει ένα σύντομο μήνυμα
λάθους και να τερματίζει.
Παραδείγματα προγραμμάτων της Γλώσσας
Λάθος πρόγραμμα 1:
int method1(int a)
{
int b;
b = a+10;
return b;
}
int main()
{
return method1(5+);
}
Λάθος πρόγραμμα 2:
int method1(int a)
{
int b;
b = ((a+10)-10)*5)/5;
return b;
}
int main()
{

int a=5;
return method1(a);
}
Σωστό πρόγραμμα 1:
int method1(int a)
{
int b;
b=0;
while( a>0)
{
b = (a+10);
a = (a-1);
break;
}
return b;
}
int main()
{
return method1(5);
}

int method1(int a)
{
int b;
b = ((a+10)-10)*5/5;
return b;
}
int main()
{
int a=5;
return method1(a);
}

int method1(int a)
{
int b;
b = a+10;
return b;
}
int method2(int c, int d)
{

e = e + d;
return e;
}

{

Παραγωγή κώδικα στη γλώσσα - στόχο
Στην περίφημη σειρά βιβλίων “The Art of Computer Programming”, που δημοσίευσε ο D.Knuth,
χρησιμοποιείται ένας ιδεατός υπολογιστής που αποκαλείται MIX. Ο MIX διαθέτει γλώσσα μηχανής
και συμβολικής αναπαράστασης (assembly) των εντολών της.
Η αρχιτεκτονική του MIX είναι μία απλοποιημένη εκδοχή της σχεδίασης των πραγματικών CISC CPUs
και η συμβολική γλώσσα του, που λέγεται MIXAL, παρέχει ένα σύνολο βασικών εντολών, που είναι
πολύ οικείες σε όποιον έχει εμπειρία προγραμματισμού σε γλώσσα assembly. Ο ορισμός του MIX και
της MIXAL είναι αρκετά ισχυρός και πλήρης, ώστε να παρέχει μία ιδεατή πλατφόρμα ανάπτυξης
σύνθετων προγραμμάτων, αρκετά κοντά στους πραγματικούς υπολογιστές, χωρίς όμως την
πολυπλοκότητα της αρχιτεκτονικής αυτών. Για αυτό το λόγο εξάλλου ο MIX και η MIXAL
χρησιμοποιούνται ευρέως ως εκπαιδευτικό περιβάλλον εκμάθησης προγραμματιστικών τεχνικών.
Ένα αρκετά ώριμο project με εργαλεία ανάπτυξης για τη γλώσσα MIXAL είναι το GNU MIX
Development Kit (διατίθεται και virtual machine). Αναλυτική περιγραφή της αρχιτεκτονικής της
μηχανής MIX και της γλώσσας MIXAL θα βρείτε στο online βιβλίο
http://www.jklp.org/profession/books/mix/index.html και στον οδηγό (manual), που διατίθεται στη

διεύθυνση http://www.gnu.org/software/mdk/manual/mdk.pdf. Στην ιστοσελίδα https://www.mix-
emulator.org/#/about διατίθεται ένας online emulator εκτέλεσης προγραμμάτων MIXAL.

Για τη δημιουργία κώδικα κλήσης υπορουτίνων (subroutines) μπορείτε να συμβουλευτείτε το
Κεφάλαιο 6 του προαναφερόμενου online βιβλίου και τον Αλγόριθμο 12.2 του συγγράμματος
«Γλώσσες Προγραμματισμού & Μεταγλωττιστές».
Ο μεταγλωττιστής σας θα πρέπει να δημιουργεί κώδικα MIXAL, που θα μπορεί να εκτελείται σε
κάποιο virtual machine ή emulator της MIXAL.
Πρέπει να τονιστεί ότι η γλώσσα MIXAL υλοποιείται στα προαναφερόμενα εργαλεία με μικρές
διαφορές (π.χ. στον ορισμό των σταθερών string) και γι ́ αυτό πρέπει ο κώδικας που θα δημιουργήσει
ο μεταγλωττιστής σας να συμμορφώνεται με τις ιδιαιτερότητες της μηχανής MIX που χρησιμοποιείτε.
Συμβουλευτείτε για το σκοπό αυτό τα παραδείγματα των MIXAL προγραμμάτων που συνοδεύουν
την πλατφόρμα MIX που έχετε επιλέξει.
